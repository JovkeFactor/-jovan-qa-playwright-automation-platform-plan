05 — Environments

Staging vs Production Strategy

Different test suites, frequencies, and policies per environment. Production is always read-only — no destructive actions.

🔵 Staging Environment
Full regression — every deploy + nightly
Smoke tests — every deploy
Critical path — nightly
Dedicated test accounts + seed data
2 retries per test
Failure alert level: Warning
🔴 Production Environment
Full regression — NOT run (too risky)
Smoke tests — after every production deploy
Critical path — nightly (read-only flows only)
No writes, no test data mutation
1 retry per test (reduce noise)
Failure alert level: Critical — immediate

11 — AI Integration

AI Integration Points

Claude powers five distinct QA assistance workflows — from automated triage on every failure to on-demand test generation from Jira tickets.

🔍 Failure Triage

After every failure, Claude reads the error, screenshot, trace, and DOM snapshot to suggest the root cause and a fix. Runs automatically via Claude Sonnet.

🔗 Self-Healing Locators

When a locator is not found, Claude compares the old selector to the current DOM and suggests a new selector. Triggers on TimeoutError.

✍️ Test Generation

Given a Jira ticket URL or user story text, Claude Opus generates a full Playwright test scaffold with page objects and assertions.

📉 Flakiness Detection

Nightly analysis of historical results DB. Claude identifies tests that frequently pass on retry, clusters the patterns, and reports root causes.

💬 NL Dashboard Query

Ask in plain English: "Which tests failed most in Shop B this week?" Claude queries the results DB and returns a human-readable answer.

Model Selection Strategy

⚡
Claude Sonnet 4.5
Day-to-day triage, self-healing locators, flakiness analysis. Runs on every failure automatically. Best cost/performance ratio.
🧠
Claude Opus 4
Complex test generation from Jira specs, deep root cause reasoning. On-demand only — triggered manually to control cost.
🏃
Claude Haiku
High-volume lightweight tasks at scale — classifying error types across hundreds of results. Cheapest option.
🏠
Ollama + Llama 3
Self-hosted, free. Only if test data (screenshots, DOM) is sensitive and cannot leave the network.
